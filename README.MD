## masoho

シンプルな **Go 言語製 REST API サーバ** のサンプルプロジェクトです。  
`net/http` と標準ライブラリのみを利用して、基本的なエンドポイント構成とディレクトリ分割を示します。

---

## 特徴

- **Go 標準ライブラリのみ**で実装
- `/health` によるヘルスチェック
- `/users` によるダミーユーザ一覧取得
- ハンドラを `handlers` パッケージに分離した、シンプルな構成

---

## 動作環境

- **Go**: 1.23 以降（`go.mod` に準拠）
- OS: macOS / Linux / Windows など、Go が動作する環境

---

## セットアップ

```bash
git clone <このリポジトリのURL>
cd masoho

# 依存関係取得（標準ライブラリのみだが念のため）
go mod tidy
```

---

## 起動方法

```bash
cd masoho
go run ./...
```

デフォルトでは `:8080` ポートで待ち受けます。

---

## エンドポイント

- **GET `/health`**
  - 用途: アプリケーションのヘルスチェック
  - レスポンス例:
    ```json
    {"status":"ok"}
    ```

- **GET `/users`**
  - 用途: ダミーユーザ一覧の取得
  - レスポンス例:
    ```json
    [
      {"id":1,"name":"Alice"},
      {"id":2,"name":"Bob"}
    ]
    ```

---

## ディレクトリ構成

```text
.
├── go.mod
├── main.go           # エントリポイント（サーバ起動・ルーティング設定）
└── handlers
    ├── health.go     # /health ハンドラ
    └── user.go       # /users ハンドラ
```

---

## 開発の進め方（例）

- **エンドポイント追加**
  - `handlers/xxx.go` を作成し、新しいハンドラ関数を定義
  - `main.go` の `main()` 内で `http.HandleFunc` を追加
- **ロジックの分離**
  - ビジネスロジックが増えてきたら、`service/` や `repository/` ディレクトリを作成して責務を分離
